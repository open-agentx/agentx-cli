# ADR 0001: Rust Workspace Architecture

## Status

Accepted

## Context

AGX is a Rust-native rewrite of the previous `quantex-cli` command surface. The project must keep `agx` as the canonical binary, preserve agent-friendly structured output, and support npm/Bun as installation channels without making JavaScript the runtime implementation.

## Decision

The repository uses a Rust workspace rooted at `Cargo.toml` with the primary CLI crate at `crates/agx-cli`.

- `crates/agx-cli` owns the `agx` binary and user-facing command surface.
- Command parsing lives in `cli.rs`; dispatch and structured command metadata live in `commands.rs`.
- Agent catalog, inspection, lifecycle package management, execution, doctor diagnostics, config, state, locks, and self-upgrade each live in separate modules.
- Cargo artifacts are written to the project-local `target/` directory through `.cargo/config.toml`.
- npm/Bun packages under `packages/` are launch and distribution wrappers only. They locate and execute native Rust binaries.
- Release metadata is generated by `scripts/package-release.mjs` into ignored project-local `release-artifacts/`.

## Consequences

The CLI implementation remains native and testable with Cargo while still allowing npm/Bun distribution. Module boundaries are intentionally simple until shared behavior proves stable enough to extract additional crates.

Every functional slice should update OpenSpec tasks and pass:

- `cargo fmt --all -- --check`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace`
- `cargo build --workspace --release`
- `pnpm run openspec:validate`
